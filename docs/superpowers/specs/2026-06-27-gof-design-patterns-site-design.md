# Design Spec — GoF 设计模式学习网站

**Date:** 2026-06-27
**Status:** Draft (awaiting user review)
**Repo:** `https://github.com/Moon-Knight1/DesignPatterns.git`
**Deployment URL:** `https://moon-knight1.github.io/DesignPatterns/`

---

## 1. Context & goals

The repository currently holds 23 Chinese Markdown files under `theory/` (one of which, `interpreter.md`, is empty) and a complete set of accompanying PNGs under `imgs/<pattern>/`. These documents describe the 23 Gang-of-Four (GoF) design patterns. The content exists; what is missing is a friendly, accessible presentation surface.

**Goal.** Build a static, Chinese-language, single-page-application learning site that:

- Surfaces all 22 non-empty patterns (excluding `interpreter.md`).
- Groups them under the three GoF categories: 创建型 (5) / 结构型 (7) / 行为型 (10).
- Renders each pattern's Markdown body faithfully, including its local image assets.
- Adopts a claymorphism visual language with vibrant category-coded accents.
- Deploys automatically to GitHub Pages via GitHub Actions on every push to `main`.

**Non-goals.** Real progress tracking, search, dark mode, i18n, backend, SSR, user accounts, analytics, and offline/PWA support are all out of scope (see §13).

## 2. Decisions log

| # | Question | Decision | Rationale |
|---|---|---|---|
| D1 | UI language | Chinese only | Source content is in Chinese; lowest complexity. |
| D2 | Progress tracking | Static decorative SVG illustration only | Avoid LocalStorage plumbing; matches claymorphism aesthetic. |
| D3 | Testimonials block | Dropped | Doesn't fit a design-pattern learning site. |
| D4 | Catalog grouping | Three GoF categories as sections | Familiar to anyone who has read the GoF book. |
| D5 | Catalog interactivity | Cards only — no search, no sort, no filter | Simplest possible; matches demo aesthetic. |
| D6 | Architecture | Approach A — Vite + Vue 3 + TS + vue-router (hash) + markdown-it + hardcoded patterns manifest | Smallest surface area; no MD editing required; reliable on GitHub Pages. |
| D7 | Router strategy | `createWebHashHistory` | No 404.html fallback needed; survives GitHub Pages project-page quirks. |
| D8 | Image serving | `publicDir: ['public', 'imgs']` + BASE_URL-prefixed src rewrites | Keeps existing `imgs/` repo structure intact. |

## 4. Information architecture

Three routes, all Chinese, hash-based:

| Route | Page | Purpose |
|---|---|---|
| `#/` | 首页 Home | Hero, three-group catalog, decorative progress demo, footer CTA |
| `#/pattern/:slug` | 模式详情 Pattern | Renders one pattern's Markdown body with sticky TOC |
| `#/about` | 关于 About | Source attribution + tech stack + repo link |

### 4.1 Home layout (scroll-driven, single page)

1. **Hero** — H1 "23 种 GoF 设计模式" + subtitle "一份面向中文读者的交互式学习手册" + two CTAs:
   - Primary: "开始学习" → `#/pattern/singleton`
   - Secondary: "浏览全部模式" → anchor to catalog
2. **三类型分组目录** — three CategorySection blocks (创建型/结构型/行为型), each containing a responsive grid of PatternCard components. Category headers are clay-styled chips with Lucide icons.
3. **学习路径装饰区** — `<ProgressDemo />`: static SVG of 5 claymorphism dots connected by a soft tube (2 filled, 1 partial, 2 empty). No state, no JS event handlers.
4. **底部 CTA banner** — `<CtaBanner />` with "准备好开始了吗？从单例模式开始 →" linking to `#/pattern/singleton`.

### 4.2 Pattern detail layout

- **Breadcrumb** at top: 首页 / <category> / <pattern>
- **H1** with English subtitle ("单例模式 / Singleton")
- Two-column on ≥1024px:
  - Left ~70%: rendered Markdown (`<MarkdownRenderer />`)
  - Right ~30%: sticky `<PatternToc />` with auto-generated anchors
- **Footer nav**: prev / next pattern cards chained across all 22 patterns in category order (Creational 1→5 → Structural 1→7 → Behavioral 1→10)

### 4.3 About

~120-word static page covering: source (Refactoring.Guru Chinese), tech stack, repo link, contribution note.

## 5. Visual design system

### 5.1 Style: claymorphism

Pillowy 3D surfaces with paired light-and-dark shadow layers for a soft sculptured feel.

| Token | Value | Use |
|---|---|---|
| `--radius-card` | 24px | Pattern cards, hero blocks |
| `--radius-pill` | 999px | Buttons, category chips |
| `--radius-soft` | 16px | Inner sub-blocks, code blocks |
| `--shadow-clay-out` | `8px 8px 16px rgba(0,0,0,.10), -8px -8px 16px rgba(255,255,255,.85)` | Default elevated clay card |
| `--shadow-clay-out-lg` | `12px 12px 28px rgba(0,0,0,.12), -10px -10px 22px rgba(255,255,255,.9)` | Hero, primary CTA hover |
| `--shadow-clay-in` | `inset 4px 4px 10px rgba(0,0,0,.08), inset -4px -4px 10px rgba(255,255,255,.9)` | Pressed / inactive state |
| `--shadow-clay-press` | `inset 6px 6px 12px rgba(0,0,0,.12), inset -6px -6px 12px rgba(255,255,255,.95)` | Active pressed state |

### 5.2 Palette

| Token | Hex | Use |
|---|---|---|
| `--bg-cream` | `#FFF7F0` | Page background |
| `--surface-card` | `#FFFFFF` | Card surface |
| `--ink-900` | `#1F2937` | Primary text (slate-800) — 14.5:1 on cream |
| `--ink-600` | `#4B5563` | Secondary text — 7.5:1 on cream |
| `--cat-creational` | `#7DD3C0` | 创建型 mint/teal |
| `--cat-structural` | `#A78BFA` | 结构型 lavender |
| `--cat-behavioral` | `#FB923C` | 行为型 peach/coral |
| `--cta-coral` | `#F97316` | Primary CTA fill |
| `--cta-coral-soft` | `#FED7AA` | CTA hover wash |
| `--accent-yellow` | `#FCD34D` | Decorative dots, "in progress" indicator |

All category / CTA tokens exceed WCAG AA against the cream background (4.5:1 for text, 3:1 for icons/UI). Category colors are paired with the Chinese category name in text so color is never the sole carrier of meaning.

### 5.3 Typography

| Role | Family | Weight | Notes |
|---|---|---|---|
| Hero H1 | `ZCOOL KuaiLe` | 400 | Playful rounded Chinese; loaded from Google Fonts |
| Headings H2-H3 | `Noto Serif SC` | 600/700 | Strong, readable |
| Body | `Noto Sans SC` | 400 | 16px base, line-height 1.75 |
| Latin accent (English pattern names) | `Plus Jakarta Sans` | 700 | Letter-spacing +0.04em |
| Code / pseudocode | `JetBrains Mono` | 400 | Used in fenced code blocks |

Type scale: 14 / 16 / 18 / 22 / 28 / 36 / 48 px. Body line-length clamped to 65–75 CJK characters.

### 5.4 Iconography

- **Lucide** (`lucide-vue-next`), stroke 2px, rounded caps, vector.
- Category icons: 创建型 `PackagePlus`, 结构型 `Blocks`, 行为型 `Workflow`.
- Nav icons: 首页 `House`, 关于 `CircleHelp`, GitHub `Github`.
- No emoji as structural icons.

### 5.5 Animation

- Micro-interactions: 180–220ms ease-out
- Page enter: 240ms cross-fade
- Section reveal on scroll: optional IntersectionObserver, 30ms stagger
- Hover on cards: scale 1.02 + larger shadow
- Press: scale 0.98 + inset shadow
- All animations collapse to opacity-only fades under `@media (prefers-reduced-motion: reduce)`
- Animate `transform`/`opacity` only — never `width`/`height`/`top`/`left`

### 5.6 Accessibility

- Focus ring: 2px solid `--cta-coral`, 2px offset on every focusable element
- Skip-to-main link visible on first Tab
- All images have descriptive Chinese `alt` (e.g., `观察者模式示意图`)
- Icon-only buttons have `aria-label`
- Heading hierarchy is sequential; the Markdown renderer's prose CSS does not skip levels
- Color is never the only indicator (each category carries a Chinese text label + Lucide icon)

### 5.7 Responsive

- Mobile-first, breakpoints: 375 / 640 / 768 / 1024 / 1440
- Container max-width 1200px; padding 16/24/32px at the three breakpoints
- Catalog grid: 1 col < 640 / 2 col 640–1024 / 3 col 1024–1280 / 4 col ≥ 1280

## 7. Data model

### 7.1 `Pattern` interface (`src/types/pattern.ts`)

```ts
export interface Pattern {
  slug: string
  titleZh: string
  titleEn: string
  category: 'creational' | 'structural' | 'behavioral'
  summary: string
  order: number
}
```

### 7.2 Pattern manifest (`src/data/patterns.ts`)

22 entries; `interpreter` is omitted. Each category has 5–10 entries ordered within itself. A derived `categories` map exposes category metadata (Chinese label, color token, Lucide icon name, items array) for use by `CategorySection`.

```ts
export const categories: Record<PatternCategory, Category> = {
  creational: { zh: '创建型', color: 'var(--cat-creational)', icon: 'PackagePlus', items: [...] },
  structural: { zh: '结构型', color: 'var(--cat-structural)', icon: 'Blocks',      items: [...] },
  behavioral: { zh: '行为型', color: 'var(--cat-behavioral)', icon: 'Workflow',    items: [...] },
}
```

`PatternView` derives prev/next by chaining all 22 patterns in category order (Creational 1→5, then Structural 1→7, then Behavioral 1→10).

### 7.3 Markdown loader (`src/data/markdown.ts`)

```ts
const raw = import.meta.glob('/theory/*.md', {
  query: '?raw',
  import: 'default',
  eager: true,
}) as Record<string, string>

export const markdownBySlug: Record<string, string> = Object.fromEntries(
  Object.entries(raw)
    .filter(([path]) => !path.endsWith('interpreter.md'))
    .map(([path, src]) => [path.match(/\/([^/]+)\.md$/)![1], src])
)
```

Build-time eager import; 22 entries guaranteed; `interpreter` excluded.

## 8. Markdown rendering pipeline

### 8.1 `useMarkdown` composable (`src/composables/useMarkdown.ts`)

```ts
import MarkdownIt from 'markdown-it'
import anchor from 'markdown-it-anchor'
import slugger from 'github-slugger'

export function useMarkdown(source: Ref<string> | string) {
  const src = isRef(source) ? source : ref(source)
  const md = new MarkdownIt({
    html: false,
    linkify: true,
    typographer: true,
    breaks: false,
  }).use(anchor, {
    slugify: (s) => slugger.slug(s),
    permalink: anchor.permalink.linkInsideHeader({
      symbol: '#',
      placement: 'before',
      ariaHidden: true,
    }),
  })

  return computed(() => {
    // Rewrite ../imgs/... → <BASE_URL>imgs/...
    const rewritten = src.value.replaceAll(
      '](../imgs/',
      `](${import.meta.env.BASE_URL}imgs/`,
    )
    return md.render(rewritten)
  })
}
```

Notes:
- `html: false` is a security choice — all `<script>` etc. in MD is escaped
- `breaks: false` preserves paragraph semantics; Chinese text doesn't need line-break coercion
- `markdown-it-anchor` uses `github-slugger` which handles CJK by transliterating per CommonMark
- No syntax highlighting (no Prism); code blocks styled by `prose.css` with `JetBrains Mono`

### 8.2 `useToc` composable

Parses the rendered HTML string with `DOMParser` (browser-only) and returns an array of `{ level, id, text }` for `<h2>` and `<h3>` nodes that carry an `id`.

### 8.3 Image src rewrite

Hash routing means the browser sees the page URL as `/DesignPatterns/` regardless of the route. A literal `../imgs/foo.png` reference in the Markdown would resolve to `/imgs/foo.png` (one level up from the site root) — wrong. We rewrite every `](../imgs/...)` to `](<BASE_URL>imgs/...)` before parse, using `import.meta.env.BASE_URL` which equals `/DesignPatterns/` in production and `/` in dev.

## 9. Routing

### 9.1 `src/router/index.ts`

```ts
import { createRouter, createWebHashHistory } from 'vue-router'

const router = createRouter({
  history: createWebHashHistory(import.meta.env.BASE_URL),
  routes: [
    { path: '/',              component: () => import('@/views/HomeView.vue'),   name: 'home' },
    { path: '/pattern/:slug', component: () => import('@/views/PatternView.vue'), name: 'pattern', props: true },
    { path: '/about',         component: () => import('@/views/AboutView.vue'),   name: 'about' },
    { path: '/:pathMatch(.*)*', redirect: '/' },
  ],
  scrollBehavior(to, _from, savedPosition) {
    if (savedPosition) return savedPosition
    if (to.hash) return { el: to.hash, behavior: 'smooth' }
    return { top: 0 }
  },
})
```

Lazy-loaded route components keep initial bundle small. Hash mode survives GitHub Pages project-page routing (no 404.html fallback required).

### 10.4 GitHub Actions workflow (`.github/workflows/deploy.yml`)

```yaml
name: Deploy to GitHub Pages

on:
  push:
    branches: [main]
  workflow_dispatch:

permissions:
  contents: read
  pages: write
  id-token: write

concurrency:
  group: pages
  cancel-in-progress: true

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with: { node-version: '20', cache: 'npm' }
      - run: npm ci
      - run: npm run build
        env: { NODE_ENV: production }
      - uses: actions/configure-pages@v5
      - uses: actions/upload-pages-artifact@v3
        with: { path: ./dist }

  deploy:
    needs: build
    runs-on: ubuntu-latest
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    steps:
      - id: deployment
        uses: actions/deploy-pages@v4
```

### 10.5 One-time repo setup

1. Repo **Settings → Pages → Build and deployment → Source**: choose **"GitHub Actions"**.
2. Repo **Settings → Actions → General → Workflow permissions**: select **"Read and write permissions"** and check **"Allow GitHub Actions to create and approve pull requests"** if not already. Without this, the `pages: write` and `id-token: write` permissions declared in the workflow can fail with a 403 on first deploy.

After that, every push to `main` triggers a build + deploy. Manual rebuild is available via the Actions tab's `workflow_dispatch` button.

## 12. Edge cases & fallbacks

| Edge | Handling |
|---|---|
| `interpreter.md` is empty | Filtered in `markdown.ts`; not in manifest; catalog shows 22 cards |
| Hash router on Pages | `index.html` is the only file Pages serves for any path; client-side routing handles the rest — no 404.html needed |
| Missing image for a pattern | `<img>` shows a claymorphism-styled "示意图" placeholder block (visible text fallback, not a broken icon) |
| Refresh on a pattern page | Hash routing → `/#/pattern/<slug>` survives refresh natively |
| Markdown with raw HTML | `html: false` in markdown-it config — any `<script>` etc. is escaped to text |
| Large `imgs/` directory | Vite copies as-is; below-the-fold images use `loading="lazy"` |
| CJK heading slugs | `github-slugger` transliterates per CommonMark rules; collisions handled by the slugger's internal counter |
| Bogus `#/pattern/<unknown>` deep-link | `PatternView` checks `markdownBySlug[slug]`; if missing, renders a claymorphism "未找到该模式" message with a link back to home |
